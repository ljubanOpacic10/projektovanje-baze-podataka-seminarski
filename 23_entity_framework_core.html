Uvod u Entity Framework Core

U prethodnoj lekciji uporedili smo Database-First i Code-First pristup i videli da Code-First omogućava da strukturu baze definišemo kroz klase u programskom jeziku. Ali da bi to funkcionisalo, potreban nam je konkretan alat koji će naše klase pretvoriti u tabele i omogućiti rad sa podacima. Taj alat je Entity Framework Core.

Šta je Entity Framework Core

Entity Framework Core, skraćeno EF Core, je ORM alat koji je razvio Microsoft za rad sa bazama podataka u C# projektima. To je najkorišćeniji ORM u .NET svetu i dostupan je besplatno. Kada kažemo da je EF Core ORM, to znači da radi tačno ono o čemu smo pričali u prethodnim lekcijama: povezuje klase iz koda sa tabelama u bazi i automatski generiše SQL komande umesto nas. EF Core podržava više sistema za upravljanje bazama podataka, uključujući Microsoft SQL Server koji smo koristili tokom kursa. To znači da možemo da nastavimo sa istim serverom, ali sada ćemo mu pristupati kroz kod umesto kroz SSMS.

Kako EF Core funkcioniše

Da bismo razumeli kako EF Core radi, treba da upoznamo dva ključna pojma: DbContext i DbSet. DbContext je klasa koja predstavlja vezu između naše aplikacije i baze podataka. Možemo da je zamislimo kao konekciju ka bazi. Kroz DbContext šaljemo sve zahteve: čitanje, dodavanje, izmenu i brisanje podataka. DbSet je svojstvo unutar DbContext-a koje predstavlja jednu tabelu u bazi. Svaki DbSet odgovara jednoj tabeli. Ako imamo tabelu Ucenik, imamo i DbSet za učenike. Pojednostavljeno:
EF Core pojam Odgovara u bazi Analogija
DbContext Cela baza podataka Konekcija ka bazi
DbSet Jedna tabela Kolekcija objekata jednog tipa

Primer: DbContext za školski sistem

Pogledajmo kako izgleda DbContext za naš školski sistem. Pretpostavimo da za sada imamo samo dve tabele: Ucenik i Profesor. Prvo definišemo klase koje predstavljaju tabele: public class Ucenik { public int Id { get; set; } public string Ime { get; set; } public string Prezime { get; set; } public string Razred { get; set; } } public class Profesor { public int Id { get; set; } public string Ime { get; set; } public string Prezime { get; set; } } Zatim pravimo DbContext klasu koja objedinjuje sve tabele: public class SkolaContext : DbContext { public DbSet<Ucenik> Ucenici { get; set; } public DbSet<Profesor> Profesori { get; set; } } SkolaContext nasleđuje klasu DbContext, čime dobija sve mogućnosti za rad sa bazom. Dva DbSet svojstva govore EF Core-u da u bazi treba da postoje tabele Ucenici i Profesori.

Šta DbContext radi u pozadini

Kada u kodu napišeš nešto poput: var ucenici = context.Ucenici.ToList(); EF Core u pozadini generiše SQL upit: SELECT * FROM Ucenici; I šalje ga bazi podataka. Rezultat koji se vrati iz baze EF Core automatski pretvara u listu objekata klase Ucenik. Ti nikada ne vidiš taj SQL upit, ali on postoji. Na isti način, kada dodaš novi objekat: context.Ucenici.Add(new Ucenik { Ime = "Ana", Prezime = "Petrovic", Razred = "II2" }); context.SaveChanges(); EF Core generiše INSERT komandu i šalje je bazi: INSERT INTO Ucenici (Ime, Prezime, Razred) VALUES ('Ana', 'Petrovic', 'II2');

Struktura jednog EF Core projekta

U pravom projektu, EF Core se koristi unutar C# aplikacije. Osnovni delovi tog projekta su: Model klase. To su klase poput Ucenik i Profesor koje predstavljaju tabele. DbContext klasa. To je klasa poput SkolaContext koja objedinjuje sve modele i omogućava komunikaciju sa bazom. Konfiguracija veze. To su podešavanja koja govore EF Core-u gde se nalazi baza podataka, na primer na kom serveru i pod kojim imenom. Konfigurisanje veze ka SQL Server-u izgleda ovako: protected override void OnConfiguring(DbContextOptionsBuilder options) { options.UseSqlServer("Server=localhost;Database=Skola;Trusted_Connection=True;"); } Ovaj tekst u zagradi naziva se connection string i sadrži informacije o serveru i bazi podataka. U ovom primeru, EF Core se povezuje na lokalni SQL Server i koristi bazu pod nazivom Skola.
Učenici se ponekad zbune jer misle da moraju da nauče potpuno novi alat od nule. U stvari, EF Core koristi koncepte koje već znate: tabele, kolone, ključeve i relacije. Razlika je samo u tome što se sada te stvari izražavaju kroz C# kod umesto kroz SQL. Ako razumeš bazu podataka, EF Core će ti brzo postati jasan.

Vizuelni pregled: od klase do baze

Hajde da rezimiramo ceo tok kojim EF Core radi sa našim školskim sistemom: Ti napišeš klasu Ucenik u C#. EF Core prepozna tu klasu kroz DbSet svojstvo u DbContext-u. Kada pokreneš migraciju, EF Core generiše SQL za kreiranje tabele. Baza se kreira ili ažurira automatski. Kada radiš sa podacima, EF Core prevodi tvoj kod u SQL komande. Ovaj ceo proces dešava se automatski, a ti ne moraš da napustiš okruženje svog programskog jezika. Šta misliš, koliko je lakše raditi kada imaš jednu klasu koja se menja na jednom mestu, umesto da ažuriraš SQL skripte, zatim menjač kod aplikacije, pa proveravaš da li je sve usklađeno?
