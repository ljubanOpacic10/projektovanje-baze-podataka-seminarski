Rad sa podacima kroz ORM

U prethodnoj lekciji naučili smo kako se pomoću migracija kreira baza podataka iz koda. Sada imamo bazu sa tabelama, ali ona je prazna. U ranijim lekcijama koristili smo SQL komande INSERT, SELECT, UPDATE i DELETE za rad sa podacima. Sada ćemo videti kako se iste operacije rade kroz EF Core, koristeći C# kod.

Četiri osnovne operacije

Rad sa podacima u bilo kojoj bazi svodi se na četiri osnovne operacije koje se često označavaju skraćenicom CRUD:
Operacija Značenje SQL komanda EF Core metod
Create Dodavanje podataka INSERT Add
Read Čitanje podataka SELECT ToList, Find, Where
Update Izmena podataka UPDATE Promena svojstva + SaveChanges
Delete Brisanje podataka DELETE Remove
Za sve primere koristićemo SkolaContext koji smo definisali u ranijim lekcijama: using (var context = new SkolaContext()) { // ovde radimo sa podacima } Ovaj blok koda otvara vezu ka bazi i na kraju je automatski zatvara. To je standardan način rada sa DbContext-om.

Dodavanje podataka (Add)

U SQL-u smo podatke unosili ovako: INSERT INTO Ucenik (ime, prezime, razred) VALUES ('Marko', 'Markovic', 'I1'); U EF Core-u isti posao radimo kroz C# kod: using (var context = new SkolaContext()) { var ucenik = new Ucenik { Ime = "Marko", Prezime = "Markovic", Razred = "I1" }; context.Ucenici.Add(ucenik); context.SaveChanges(); } Šta se ovde dešava? Kreiramo objekat klase Ucenik i postavljamo mu svojstva. Metodom Add dodajemo ga u DbSet. Ali to samo govori EF Core-u da treba da ga zapamti. Podatak se zaista upisuje u bazu tek kada pozovemo SaveChanges. SaveChanges je komanda koja šalje sve pripremljene promene u bazu. Ako zaboraviš da je pozoveš, ništa neće biti sačuvano.

Čitanje podataka (LINQ)

U SQL-u smo podatke čitali pomoću SELECT komande. U EF Core-u koristimo LINQ, koji je ugrađen u C# jezik i omogućava pretragu podataka na veoma čitljiv način. Čitanje svih učenika: // SQL: SELECT * FROM Ucenici var sviUcenici = context.Ucenici.ToList(); Čitanje učenika iz određenog razreda: // SQL: SELECT * FROM Ucenici WHERE Razred = 'I1' var ucenici = context.Ucenici .Where(u => u.Razred == "I1") .ToList(); Pronalaženje jednog učenika po Id-u: // SQL: SELECT * FROM Ucenici WHERE Id = 1 var ucenik = context.Ucenici.Find(1); Čitanje samo imena i prezimena: // SQL: SELECT Ime, Prezime FROM Ucenici var imena = context.Ucenici .Select(u => new { u.Ime, u.Prezime }) .ToList(); Primeti kako je svaki LINQ upit veoma sličan odgovarajućem SQL upitu. Where odgovara WHERE uslovu, Select odgovara listi kolona, a ToList izvršava upit i vraća rezultate.

Čitanje povezanih podataka

Jedna od velikih prednosti ORM-a je lak rad sa povezanim podacima. Na primer, ako želimo da učitamo učenika zajedno sa njegovim ocenama: // učitaj učenika i njegove ocene var ucenik = context.Ucenici .Include(u => u.Ocene) .FirstOrDefault(u => u.Id == 1); Metod Include govori EF Core-u da pored učenika učita i njegove ocene u jednom koraku. Bez Include, ocene ne bi bile automatski učitane. Za istu stvar u SQL-u bi morali da pišemo JOIN: SELECT u.*, o.* FROM Ucenici u JOIN Ocene o ON u.Id = o.UcenikId WHERE u.Id = 1; EF Core ovo radi umesto nas.

Izmena podataka (Update)

U SQL-u smo podatke menjali ovako: UPDATE Ucenik SET razred = 'II1' WHERE id_ucenik = 1; U EF Core-u, najpre pronađemo objekat, promenimo mu svojstvo i sačuvamo: using (var context = new SkolaContext()) { var ucenik = context.Ucenici.Find(1); ucenik.Razred = "II1"; context.SaveChanges(); } EF Core prati sve promene na objektima. Kada pozoveš SaveChanges, on sam prepozna šta je promenjeno i generiše odgovarajuću UPDATE komandu.

Brisanje podataka (Remove)

U SQL-u smo brisali ovako: DELETE FROM Ucenik WHERE id_ucenik = 1; U EF Core-u: using (var context = new SkolaContext()) { var ucenik = context.Ucenici.Find(1); context.Ucenici.Remove(ucenik); context.SaveChanges(); } Prvo pronađemo objekat, zatim ga uklonimo iz DbSet-a pomoću Remove, i na kraju sačuvamo promene. EF Core generiše DELETE komandu.

Uporedni pregled: SQL vs EF Core

Hajde da na jednom mestu uporedimo sve četiri operacije:
Operacija SQL EF Core
Dodavanje INSERT INTO ... VALUES ... Add + SaveChanges
Čitanje svih SELECT * FROM ... .ToList()
Filtriranje SELECT ... WHERE ... .Where(...).ToList()
Izmena UPDATE ... SET ... WHERE ... Promena svojstva + SaveChanges
Brisanje DELETE FROM ... WHERE ... Remove + SaveChanges
Najčešća greška kod učenika je da zaborave da pozovu SaveChanges nakon što dodaju, izmene ili obrišu podatke. Bez te komande, promene ostaju samo u memoriji i nikada se ne primenjuju na bazu. Svaki put kada menjaš podatke, na kraju mora da stoji SaveChanges.

Primer: rad sa ocenama učenika

Pogledajmo malo složeniji primer. Želimo da dodamo ocenu učeniku i zatim prikažemo sve ocene za tog učenika: using (var context = new SkolaContext()) { // dodajemo ocenu učeniku sa Id = 1 iz predmeta sa Id = 1 var ocena = new Ocena { Vrednost = 5, Datum = DateTime.Now, UcenikId = 1, PredmetId = 1 }; context.Ocene.Add(ocena); context.SaveChanges(); // čitamo sve ocene za učenika sa Id = 1 var ocene = context.Ocene .Where(o => o.UcenikId == 1) .ToList(); } Primeti kako koristimo strane ključeve (UcenikId i PredmetId) na isti način kao u SQL-u, ali bez pisanja SQL-a.
